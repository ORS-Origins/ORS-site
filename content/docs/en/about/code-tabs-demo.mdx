This page demonstrates the code block switching experience of "same logic, multiple languages". Both `Tabs` use `groupId="language"` and `persist`, so switching one group will sync the other, and the selection is saved to `localStorage` — it will be remembered even after closing and reopening the browser.

## Sorting Algorithm

<Tabs groupId="language" persist items={['C++', 'Python', 'Java']}>

<Tab value="C++">

```cpp
// src/algo/sort.cpp
#include <algorithm>
#include <vector>

void sort_asc(std::vector<int>& nums) {
  std::sort(nums.begin(), nums.end());
}
```

</Tab>

<Tab value="Python">

```python
# src/algo/sort.py
def sort_asc(nums: list[int]) -> list[int]:
    return sorted(nums)
```

</Tab>

<Tab value="Java">

```java
// src/algo/Sort.java
import java.util.Arrays;

class Sort {
  static int[] sortAsc(int[] nums) {
    Arrays.sort(nums);
    return nums;
  }
}
```

</Tab>

</Tabs>

## Hello World

The group below only has two languages. When switching the language in the group above, this group should sync accordingly; if Java is selected above and this group doesn't have Java, it will keep its own default option.

<Tabs groupId="language" persist items={['C++', 'Python']}>

<Tab value="C++">

```cpp
// src/hello.cpp
#include <iostream>
int main() {
  std::cout << "Hello, Neoverse!\n";
  return 0;
}
```

</Tab>

<Tab value="Python">

```python
# src/hello.py
print("Hello, Neoverse!")
```

</Tab>

</Tabs>
